# セキュリティチェック

このプロジェクトのセキュリティ観点を確認してください。特に以下の点を重点的にチェック：

## 主要な確認項目

### 1. データ送信先の確認
- **Google API以外へのデータ送信がないか**を最優先で確認
- すべてのネットワークリクエストを洗い出し、送信先ドメインを特定
- 動画ファイル、APIキー、ユーザー設定、スクリーンショット、生成されたドキュメントがどこに送信されているかを確認

### 2. 外部サービス・トラッキングの確認
- Google Analytics、Sentry、その他の分析・監視サービスの使用状況
- CDNからのスクリプト読み込み
- サードパーティAPIの呼び出し

### 3. 依存関係のセキュリティ
- `package.json`の依存パッケージで疑わしいもの
- 未使用の依存パッケージ

### 4. データ永続化のセキュリティ
- LocalStorageに保存されるデータ（特にAPIキー）
- IndexedDBに保存される動画ファイル
- データのクリア機能が適切に実装されているか

### 5. コード内のセキュリティリスク
- XSS（クロスサイトスクリプティング）の脆弱性
- プロトタイプ汚染の可能性
- 安全でない正規表現

## チェック手順

1. `src/services/` 以下のすべてのファイルでネットワークリクエストを検索
2. `fetch()`、`XMLHttpRequest`、外部APIクライアントの使用箇所を特定
3. 各リクエストの送信先URLとデータ内容を確認
4. `package.json` の依存関係を確認
5. セキュリティ上の問題があれば、具体的な指摘と改善提案を提示

## 期待する出力

以下の形式でレポートを作成：

```markdown
# セキュリティチェックレポート

## データ送信先の確認

### Google API へのリクエスト
- [送信先URL] - [送信データの種類] - [ファイル名:行番号]

### その他の外部サービス
- なし / あり（詳細を記載）

## 検出された問題

### 🔴 重大な問題
（あれば記載）

### 🟡 改善推奨事項
（あれば記載）

### ✅ 問題なし
（確認済み項目）

## 推奨事項
（セキュリティ向上のための提案）
```

ユーザーが理解しやすいように、具体的なファイル名と行番号を含めて報告してください。
