---
description: ビルドとテストを実行し、エラーやWarningを解決する
---

プロジェクトのビルドとテストを実行し、発見されたエラーやWarningを分析して解決してください。

以下の手順で実行してください：

## 1. テストの実行と確認

`npm test` を実行して、テスト結果を確認してください。

- テストが失敗している場合：
  - 失敗しているテストファイルと具体的なエラーメッセージを確認
  - エラーの原因を分析（コードの変更、依存関係の問題、型エラーなど）
  - 適切な修正を実施

- Warningがある場合：
  - Warningの種類と原因を特定
  - 可能であれば修正（非推奨APIの使用、型の不一致など）

## 2. ビルドの実行と確認

`npm run build` を実行して、ビルド結果を確認してください。

- TypeScriptのコンパイルエラーがある場合：
  - エラーの詳細を確認（ファイル名、行番号、エラー内容）
  - 型エラー、importエラー、構文エラーなどを修正

- Viteビルドのエラーがある場合：
  - バンドルエラー、依存関係の問題を解決

- Warningがある場合：
  - チャンクサイズの警告：
    - 500KB以上のチャンクがある場合は動的インポートやcode-splittingを検討
    - ただし、既存の機能を壊さないように注意
  - 循環依存の警告：
    - import構造を見直して循環を解消
  - その他のWarning：
    - 原因を特定して適切に対処

## 3. 修正の検証

エラーやWarningを修正した後、以下を再度実行して確認してください：

1. `npm test` - すべてのテストがパスすることを確認
2. `npm run build` - ビルドが成功することを確認

## 4. 修正内容の報告

修正を完了したら、以下を報告してください：

- 発見されたエラー/Warningの数と種類
- 実施した修正の概要
- テストとビルドの最終結果
- 修正できなかった問題がある場合はその理由

## 注意事項

- **既存の機能を壊さない**: 修正によって既存のテストが失敗しないように注意
- **段階的な修正**: 複数の問題がある場合は、一つずつ修正して確認
- **コミットは不要**: このコマンドでは修正のみを行い、コミットは行わない
- **CLAUDE.mdに従う**: プロジェクトの重要なパターンとコーディング規約を遵守
